# Framework — Value Equation (Hormozi)

## Origin
Alex Hormozi, *$100M Offers* (2021). The single most-cited offer-design
framework of 2022-2026. Used to engineer offers that "make people feel
stupid saying no."

## When to use
- Designing or pricing a new product/service
- An existing offer is converting poorly despite traffic + interest
- You want to compare your offer head-to-head against a competitor's
- Considering a pivot in value-capture (PRICING + offer change together)

## When NOT to use
- Pre-validation. If the customer/problem isn't proven yet, fix that
  first (CUSTOMER_DISCOVERY).
- Pure commodity products where price is the only lever.

## The formula

```
            Dream Outcome × Perceived Likelihood of Achievement
Value =  ────────────────────────────────────────────────────────
                    Time Delay × Effort & Sacrifice
```

**Numerator** — what raises perceived value:
1. **Dream outcome** — the *experience* the buyer wants (Hawaii, not
   the plane). Sell the destination, not the vehicle.
2. **Perceived likelihood of achievement** — "Will it actually work for
   me?" Built via proof, guarantees, social proof, demo'd outcomes.

**Denominator** — what kills perceived value:
3. **Time delay** — gap between purchase and outcome. Make it feel
   instant, smooth, automatic.
4. **Effort & sacrifice** — what the buyer must DO. Reduce, automate,
   or do-for-them.

## The strategic insight

> Most businesses optimize the numerator. The real magic is in the
> denominator.

Cutting time-to-value in half ≈ doubling perceived value. Removing
3 steps of friction ≈ same effect. Stack reductions on the denominator
beat boosting the numerator alone.

## Step-by-step application

### 1. Score the four components today (1-10)
For your current offer, score each component as the customer would
perceive it:
- Dream outcome clarity: __/10
- Perceived likelihood: __/10
- Time delay (10 = instant): __/10
- Effort & sacrifice (10 = effortless): __/10

### 2. Identify the weakest component
Multiply numerator scores; divide by denominator product. Whichever
component is lowest (or highest, on the denominator) is your highest-
leverage fix.

### 3. List 3-5 levers for the weakest component
Examples:
- Dream outcome: clearer outcome statement, vivid case studies, sharper
  positioning of the *experience*
- Likelihood: money-back guarantee, milestone-based pricing, proof
  artifacts (logos, ROI calcs, before/after data)
- Time delay: automation, templates, done-for-you tier, instant-onboarding
- Effort: prefilled defaults, smart guidance, agent-assisted setup, one-
  click integrations

### 4. Pick the one lever to ship next
Constraint-based. What can ship in 30 days that moves that one component
2 points up (or down for denominator)?

## Output structure when running this on a project

```
## Value Equation — {project name}

### Current offer scores (subjective, validate via 5-10 customer convos)
| Component | Score | Justification |
| Dream outcome | x/10 | ... |
| Perceived likelihood | x/10 | ... |
| Time delay (10 = instant) | x/10 | ... |
| Effort & sacrifice (10 = effortless) | x/10 | ... |

### Weakest component
{Component} → score {x}/10. Cost in perceived value: roughly {Y}x.

### Top 3 levers to fix it
1. {lever} — ship-cost: {S/M/L}
2. {lever} — ship-cost: {S/M/L}
3. {lever} — ship-cost: {S/M/L}

### Recommended next move
Ship lever #{N} in {30/60/90} days. Re-survey 5 customers. If component
moves +2, double down. If <+1, try lever #{M}.
```

## Decision-log entry suggestion
- Category: pricing or product
- Reversibility: reversible (most offer tweaks are)
- Pre-mortem: "Wrong if customers actually want a different dream
  outcome than we're selling"
- Revisit: 60 days post-launch with conversion delta
